hexo与github的关联操作及部署

1.SSH key 的生成,生成过程中不设置密码，后续提交项目就不需要验证密码，生成的ssh key在根目录下的.ssh文件夹下的id_rsa.pub文件。

2.将生成的ssh key复制粘贴到https://github.com/settings/ssh的new SSH key中,通过如下命令检查是否配对成功。

3.再配置用户名和邮箱，git代码提交需要通过这两项来进行判断

4.在Hexo初始化生成的文件中的_config.yml文件中，找到Deployment字段，进行配置修改

5.完成后可以先通过本地服务，检查下效果，确认ok后，再进行部署到github的操作,在部署上github时最好先清除下hexo的缓存。

最后只要打开查看即大功告成。

附

1.如果上面的操作失败,需要提前安装一个插件。

2.如果执行hexo d操作后出现error deployer not found:github的错误提示，则说明_config.yml的文件配置出错了。
原因：因为使用的是hexo3.0版本，所以需要做三件事情

  • 安装 npm install hexo-deployer-git –save
  • 将deploy 的 type由github改为git
  • 将repository的https改为http
